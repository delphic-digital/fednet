Skeletor

Skeletor is a mobile first Front End Developer boilerplate.
Current version is 1.0.1
Download on github

Skeletor is designed to be as flexible with designs as possible, so the main goal was to not lock the developer into a grid or template system. It provides organization through a file structure standard and process tools.

Features

  • Grunt tasks for environment switching
  • Sass with handy mixins
  • Zepto.js library with a jQuery fallback.
  • onMediaQuery for responsive JavaScript development.
  • LABjs (Loading And Blocking JavaScript) performance script loader with custom Delphic wrapper and config.

Get Started

Skeletor requires node.js and grunt. So get those.

JavaScript

These are included by default.

  • js/config.js Configure sites JavaScript plugins paths. Part of delphic.inject .
  • js/main.js Contains all the site JavaScript application code. ALL JavaScript application code should go into this one file for performance and maintainability.
  • js/vendor/LAB.js LABjs JS script loader.
  • js/vendor/onmediaquery.js onMediaQuery Triggers JS based on media context.
  • js/vendor/zepto.js Zepto.js Lightweight jQuery alternative. Execution is a lot faster.
  • js/utilities/delphic.inject.js Wrapper function for LABjs. This is included by default because page specific JavaScripts should be loaded on demand. Docs
  • js/plugins/* Put any page specific scripts in here. There are a couple examples included.

Build Process

When your site is ready for production, you can run some grunt commands.

A standard build will:

  • Optimize JS by combining and then minify zepto.js, onmediaquery.js, LAB.js,delphic.jsload.js and main.js into one file. Minified JS plugins will also be created in the plugins folder.
grunt dev

grunt prod
